# CODEX 执行说明补充（代码卫生与署名规范）

> 目的：保证仓库内代码与注释保持“项目化、可交付、可评审”的风格，不包含与实现无关的对话信息或工具宣传语。

## 1. 代码与注释风格
- 代码/注释必须采用正式工程文风；禁止出现聊天语气、对话过程、以及与实现无关的提示语（如“由某某工具生成”等）。
- 类/方法注释聚焦：用途、输入输出、边界条件、异常与权限要求。

## 2. 署名与版权信息（仓库级）
- 仓库根目录维护 `NOTICE.md`（或 `COPYRIGHT.md`）：声明项目名称、作者（本人姓名）、学校/课程信息、年份。
- 如引用开源代码或模板，必须保留原有版权/许可声明，并在 `NOTICE.md` 中列出引用来源与许可（避免误删许可头）。

## 3. 提交记录（Git）
- Git 提交的 `user.name` / `user.email` 需设置为本人信息，确保提交者身份一致。
- 提交信息采用规范格式（如 `feat: ...` / `fix: ...` / `docs: ...`），避免出现与实现无关的描述。

## 4. 生成内容的处理原则
- 任何自动生成/外部参考的代码，合并入主分支前必须人工审阅、重构、补齐注释与测试（保证可解释、可维护）。
