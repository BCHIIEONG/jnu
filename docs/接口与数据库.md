# 接口与数据库

## 1. BOOT-001 范围说明
- 当前阶段仅落地工程底座与通用能力，不涉及业务表结构新增。
- 已实现内容：统一返回、全局异常、traceId、参数校验统一返回、健康检查、Swagger、示例接口。

## 2. 统一返回结构
- 统一字段：`code`、`message`、`data`、`traceId`、`timestamp`
- 成功示例（`GET /api/ping`）：
```json
{
  "code": 0,
  "message": "OK",
  "data": {
    "message": "pong"
  },
  "traceId": "98cb1b5076ea430a9067ce2f09bb8975",
  "timestamp": "2026-02-10T07:05:09.468053500Z"
}
```

## 3. API 清单（当前已实现）
- `GET /api/ping`：连通性检查，返回统一成功响应。
- `POST /api/ping/echo`：参数校验示例接口（`message` 非空）。
- `GET /actuator/health`：健康检查接口。
- `GET /swagger-ui/index.html`：Swagger UI 页面。
- `GET /v3/api-docs`：OpenAPI 文档 JSON。

## 4. 错误码（当前）
- `0`：成功
- `40000`：请求参数或请求体格式错误
- `40001`：参数校验失败
- `40100`：未授权（预留）
- `40300`：无权限（预留）
- `50000`：服务器内部错误
