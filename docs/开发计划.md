# 开发计划（从现在起按周滚动维护）

> 原则：**不新增需求**，先把 MVP 闭环做稳、做完整。

## 里程碑
- M2：核心业务链跑通（登录/RBAC/任务/报告提交/基础日志）
- M3：清单模块 MVP 全覆盖（签到/设备/进度/批阅/导出/查重 v1）
- M4：代码冻结 + 可稳定演示

## 近期两周（示例）
### 第1周：工程底座 + 权限
- 后端：工程骨架、统一返回、异常、JWT/RBAC、RBAC表结构
- 前端：登录、路由权限、基础布局（PC/H5 响应式）
- 输出：第一批可运行接口 + 3个角色账号

### 第2周：报告闭环 v1
- 教师：任务 CRUD
- 学生：报告提交（Markdown+附件+版本）
- 教师：批阅评分 + 导出 CSV
- 输出：端到端演示脚本
