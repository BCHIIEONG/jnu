# CODEX 执行说明（给 Codex 的固定提示词/工作方式）

## 输入优先级（必须遵守）
1) `定稿记录.txt`：范围与功能清单（不要擅自加需求）
2) `log记录/决策日志.md`：已定稿的最新决策（优先于旧内容）
3) `log记录/Codex任务单.md`：本迭代要做的工单（按 Task-ID 执行）
4) 其它 docs：技术框架/接口与数据库/部署/测试/规范

## 执行规则
- 不做“满汉全席”：每个模块只做 MVP，先跑通闭环。
- 每个任务必须：
  - 代码可运行（最小可演示）
  - 有基本测试（至少接口层 smoke）
  - 更新对应文档（接口/表结构/运行方式）
- 不允许：
  - 未经 `决策日志` 定稿就改技术路线
  - 未经确认新增业务模块

## 产出要求
- 每完成一个 Task-ID：
  1) 更新 `log记录/开发日志.md`（记录完成情况/问题/下一步）
  2) 在任务单里标记状态（进行中/完成）
  3) 如涉及接口或表结构变更，必须同步更新 `docs/接口与数据库.md`
